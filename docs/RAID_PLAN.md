## v0.1 → v0.2 주요 변경

- **D2 강화**: 유저 페널티 완전 제거 (HP 없음 + XP 차감도 없음). 보스 공격은 순수 연출.
- **D7 확정**: 하이브리드 패널 모델 — 공용 라이브 임베드 + `/레이드참가` 로 개인 ephemeral 패널(상태창 + 버튼).
- **D13 폐기**: AFK 자동 패스 시스템을 만들지 않음. 데미지 비율 정산으로 무임승차 자연 차단.
- 그 외 D1, D3~D6, D8~D12 모두 v0.1 추천대로 확정.

## 0. 한 줄 요약

> 길드 전체가 참여하는 PvE 컨텐츠. **레벨링 시스템이 입력, 보상이 출력**.
> 임베드 1개 자동 갱신으로 라이브 전투화면 + 종료 후 기여도 룰렛.

---

## 1. 핵심 목표

1. **레벨링 시스템의 종착점** — 쌓아온 레벨·스탯·아이템을 "써먹는" 자리.
2. **길드 활동성 가속기** — 매주 정해진 시간에 사람을 모으는 정기 이벤트.
3. **확률·연출로 도파민** — 결과에 약간의 운과 화려한 임베드를 섞어 "한 번 더" 유도.
4. **저사양·1015 친화** — 임베드 갱신 빈도·DB 쓰기 빈도를 명시적으로 통제.

비목표(현 단계 제외): 길드 vs 길드, 골드/상점/장비 슬롯, 캐릭터 사망 처리.

## 3. 채널 · 권한 구조

### 채널
- 이름: `보스레이드` (기본). `/레이드채널설정 <channel>` 로 변경 가능.
- 없으면 봇이 자동 생성 — 카테고리 최상단, 누구나 읽기/쓰기.
- 봇 권한 필요: `manage_channels`(자동 생성 시), `manage_messages`(고정 메시지), `embed_links`, `read_message_history`.

### 채널 사용 규칙
- 비전투 시간: 자유 채팅 허용(가볍게).
- 전투 진행 중: 라이브 임베드가 채널 맨 위에 **고정**(pinned).
- 전투 중 일반 채팅은 허용 — 분위기 띄우는 응원/조롱 OK.

### 권한 게이트
| 명령 | 대상 |
|---|---|
| `/공격`, `/스킬`, `/레이드상태` | 누구나 (해당 채널에서만) |
| `/레이드소환 <보스>` | 오너 + 서버 관리자 |
| `/레이드채널설정` | 서버 관리자 (manage_guild) |
| `/레이드일정설정` | 서버 관리자 |
| `/레이드취소` | 오너 |

            Phase(at_hp_pct=0.3, name="파괴",   bonus={"boss_atk": 2.0, "weakness": "fire"}),
        ],

        time_limit=timedelta(minutes=30),

## 5. 전투 시스템

### 5.1 데미지 공식 (초안)

```
기본_공격력  = 10 (전부 동일 출발선)
힘_보정      = 1 + STR × 0.02         # 100 STR = ×3
공격력       = 기본_공격력 × 힘_보정

치명_확률    = min(0.50, 0.05 + LUK × 0.003)
                                       # 5% base, 100 LUK = 35%, 150 LUK = 50% 캡
치명_배수    = 1.5 + AGI × 0.005      # 100 AGI = ×2.0
약점_확률    = min(0.40, INT × 0.002) # 0%, 100 INT = 20%, 200 INT = 40% 캡
약점_배수    = 2.0

최종_데미지  = 공격력
            × (치명 적중 시 치명_배수)
            × (약점 적중 시 약점_배수)
            × 페이즈_가중치           # 보스가 약점 노출했을 때 등
            × 부스트_배수             # 활성 부스트 아이템 효과
```

### 5.2 쿨다운

```
평타_쿨다운  = max(15, 60 × (1 − AGI × 0.003))
                                       # 60s base → 100 AGI 42s, 200 AGI 24s, 캡 15s
스킬_쿨다운  = 스킬마다 60~300s, AGI 보정 동일 적용
```

### 5.3 자원 — "마나"

```
최대_마나    = 100 + INT × 1
마나_재생    = (5 + INT × 0.1) / 분    # 자연 재생, 최대치까지
스킬_코스트  = 스킬별 30~80
```

마나 부족 시 스킬 사용 불가 → 평타로 시간 끌어야 함.

### 5.4 스킬 (초안 4종)

| 키 | 이름 | 스탯 | 마나 | 쿨다운 | 효과 |
|---|---|---|---:|---:|---|
| `slash` | 강타 | STR | 30 | 60s | 데미지 ×1.8, 일반 공격 1회 |
| `volley` | 화살비 | AGI | 50 | 90s | 3회 연속 공격, 각 ×0.6 데미지 |
| `pierce` | 약점간파 | INT | 60 | 120s | 약점 적중 확정, ×2.5 데미지 |
| `fortune` | 행운의 일격 | LUK | 40 | 100s | 치명 확정, ×2.0 데미지 (스택 시 ×3) |

### 5.5 보스 행동 (순수 연출)

> v0.2 결정: 유저는 페널티 0. 보스 공격은 데미지·XP 차감 없이 **로그 연출용**.

- 보스는 N초마다 임베드 로그에 한 줄 추가 (예: "화염 골렘이 포효한다 🔥", "용암을 분출한다 💥").
- 페이즈 전환 시 보스가 광폭화하면 **자신이 받는 데미지가 변경**되거나 **약점이 노출**되는 식으로 게임성 유지.
  - 예: 페이즈 2 "격노" 진입 시 보스가 받는 데미지 +20%, 약점 '화염' 노출 → INT 약점 공격 트리거됨.
- 위협(threat) 시스템 불필요. 보스가 누군가를 "공격"하는 개념 자체가 없음.
- 유일한 실패 조건 = **시간 초과**. 시간 내에 HP 를 0 까지 못 깎으면 패배 → 위로 보상만.

## 6. 진행 화면 — 하이브리드 패널 (v0.2)

> 공용 임베드 1개(모두에게 보임) + 개인 ephemeral 패널(`/레이드참가` 로 호출, 본인만 보임).
> 두 화면은 **독립적으로 갱신**되어 다른 사람 패널엔 영향 없음.

### 6.2 개인 ephemeral 패널 (`/레이드참가` 호출, 본인만)

```
┌─ 내 상태창 ──────────────────────┐
│ 화염사냥꾼  Lv 25                 │
│ 💪 STR 45  💨 AGI 30              │
│ 🧠 INT 60  🍀 LUK 15              │
│ ────────────────────────────────  │
│ 마나 ▰▰▰▰▰▰▱▱  70 / 160         │
│ 평타 쿨다운: ⚔️ 즉시              │
│ 강타  쿨다운: 💥 28초             │
│ 화살비 쿨다운: 🏹 즉시            │
│ 약점간파 쿨다운: 🎯 73초          │
│ 행운의일격 쿨다운: ✨ 즉시        │
│ ────────────────────────────────  │
│ 누적 데미지: 8,332 (#1)           │
└──────────────────────────────────┘
 [⚔️ 평타]  [💥 강타(30)]  [🏹 화살비(50)]
 [🎯 약점간파(60)]  [✨ 행운의일격(40)]
```

- 본인 보유 스킬(전부 4종)만 버튼으로 표시. 비활성(쿨다운/마나 부족) 버튼은 disabled.
- 버튼은 본인 패널에서만 동작 — `interaction.user.id` 검증 필수.
- 행동 → `interaction.response.edit_message()` 로 본인 패널 갱신 + 공용 임베드 디바운스 큐 push.

### 6.3 갱신 전략 (Cloudflare 1015 안전)

**공용 임베드**
- 하나의 메시지를 `message.edit()` 로 갱신 — 새 메시지 발송 X.
- **행동 디바운스 2초** + **하드 캡 3초 간격**.
- `GUARD.is_paused()` 시 edit 스킵, 메모리에만 누적 → 가드 해제 후 다음 edit 에 반영.
- 종료 임베드는 새 메시지로 별도 발송 (룰렛 연출에 사용).

**개인 패널**
- 클릭한 본인의 interaction 응답으로만 갱신 → **개인 패널은 그 한 사람의 API 호출 1건**.
- 다른 유저들 패널은 영향 X (각자 호출 시점에 최신 상태로 갱신).
- ephemeral interaction 만료(약 15분)를 대비:
  - 패널 timeout(15분) 도달 시 자동 disable 버튼 + 안내 "`/레이드참가` 다시 입력하세요"
  - 60분 보스 진행 중엔 유저가 4~5분마다 재입력하는 흐름 자연스러움

### 6.4 인터랙티브 선택

- **명령은 `/레이드참가` 하나만** — 모든 행동은 개인 패널의 **버튼**.
- View 타임아웃·동시성은 `interaction.user.id` 검증 + 패널 만료 안내로 단순화.
- 슬래시-only 옵션은 v0.1 추천이었으나 D7 에서 하이브리드로 확정.

## 7. 기여도 & 보상

### 7.1 기여도 계산

```
기여도(유저)  = 총_데미지(유저)
               + 스킬_사용_횟수(유저) × 50   # 단순 평타만보다 스킬 활용 가산
               + 결정타_가산                   # 마지막 일격을 친 유저에게 +500
```

### 7.2 보상 분배

| 구간 | 규칙 |
|---|---|
| **기본** | 모든 참가자에게 XP `base_xp × 0.5` 균등 지급 |
| **비례** | 기여도 비율 × `base_xp × 3` 추가 지급 |
| **TOP1** | 결정타 보너스 + `top1_extra_xp` (예: 2000) |
| **TOP3** | 보스 드롭 테이블에서 1회 굴림 (확률) |
| **참여 보상** | 모든 참가자에게 드롭 테이블 0.3 회 굴림(낮은 확률) |
| **마지막 일격** | 별도 룰렛 1회 (희귀 아이템 확률↑) |

### 7.3 보상 룰렛 연출 — **도파민 핵심**

종료 임베드에 슬롯 머신 시뮬:
```
🎰 보상 룰렛 굴리는 중...
🎰 굴리는 중...
🎰 🍀 행운의 부적!

→ 유저A 가 🍀 행운의 부적 을(를) 획득!
```

- 임베드 3회 edit (각 1초 간격) — 긴장 → 결과.
- 희귀 결과(엘릭서/행운의부적)는 별도 **이펙트** 임베드 추가 (보라색 + 반짝 이모지).
- 일반 결과는 1줄 추가만.

### 7.4 패배(시간 초과·전멸) 처리

- 패배해도 **위로 보상**: 참가자 모두 XP `base_xp × 0.2`.
- 보스는 24시간 쿨다운 후 재등장 가능.

## 8. 데이터 모델 (DB 스키마 초안)

```sql
-- 진행 중/완료된 레이드 메타
CREATE TABLE raids (
    id           BIGSERIAL PRIMARY KEY,
    guild_id     BIGINT NOT NULL,
    boss_key     TEXT NOT NULL,
    max_hp       BIGINT NOT NULL,
    current_hp   BIGINT NOT NULL,
    phase        INT NOT NULL DEFAULT 1,
    status       TEXT NOT NULL,                    -- 'active' | 'victory' | 'defeat' | 'cancelled'
    channel_id   BIGINT NOT NULL,
    message_id   BIGINT,                            -- 라이브 임베드 메시지 id
    started_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
    ended_at     TIMESTAMPTZ,
    last_action_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_raids_active ON raids (guild_id, status);

-- 참가자별 누적 (UPSERT)
CREATE TABLE raid_participants (
    raid_id      BIGINT NOT NULL REFERENCES raids(id) ON DELETE CASCADE,
    user_id      BIGINT NOT NULL,
    total_damage BIGINT NOT NULL DEFAULT 0,
    hits         INT NOT NULL DEFAULT 0,
    skills_used  INT NOT NULL DEFAULT 0,
    final_blow   BOOLEAN NOT NULL DEFAULT FALSE,
    joined_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (raid_id, user_id)
);

-- 마나/쿨다운 (메모리만? DB?) — 결정 보류, 일단 메모리 + 종료 시 정산
-- 옵션: user_combat_state(user_id, current_mana, last_attack_at, ...)

-- 행동 로그 (상세 로그/리플레이 — 선택)
CREATE TABLE raid_actions (
    id        BIGSERIAL PRIMARY KEY,
    raid_id   BIGINT NOT NULL REFERENCES raids(id) ON DELETE CASCADE,
    user_id   BIGINT,                                -- 보스 행동은 NULL
    action    TEXT NOT NULL,                          -- 'attack' | 'skill:strike' | 'boss_atk' | ...
    damage    INT,
    crit      BOOLEAN,
    weakness  BOOLEAN,
    at        TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_raid_actions_recent ON raid_actions (raid_id, id DESC);

-- 길드별 레이드 설정
ALTER TABLE guild_config ADD COLUMN raid_channel_id BIGINT;
ALTER TABLE guild_config ADD COLUMN raid_schedule TEXT; -- cron 형식 또는 'weekly:sun:18'
```

상태 복원: 봇 재시작 시 `status='active'` 인 raids 를 메모리로 로드 → 임베드 메시지 핸들 복구.

## 10. 균형·안전장치

### 10.1 Cloudflare 1015 친화
- 임베드 edit: 2초 디바운스 + 3초 하드 캡, `GUARD.is_paused()` 시 스킵
- DB 쓰기: 행동 누적은 **메모리 → 5초마다 일괄 flush**
- 종료 시점에 최종 정산만 단일 트랜잭션

### 10.2 동시성
- 한 길드에 동시 진행 레이드 **1마리** 제한 (락은 raids 테이블 unique partial index 로)
- 행동 처리는 `asyncio.Lock` 으로 직렬화 (데미지·HP 정합)

### 10.3 도배·악용 방지 (v0.2 — AFK 시스템 폐기)
- 평타·스킬 쿨다운으로 도배 차단 (행동 자체가 일정 간격 강제)
- **AFK 자동 패스 시스템 없음** — D13 결정: 데미지 비율로 정산하므로 무임승차 자연 차단
  - 행동 0회 = 데미지 0 = **비례 보상 0** (위로 보상만 받음)
  - 1번만 행동해도 그 데미지만큼 정산 → 트롤도 부작용 없음
  - 별도 AFK 감지·타이머·통보 로직 불필요 → 구현·운영 단순화
- 1보스에 한 유저가 가져갈 수 있는 보너스 보상은 별도 상한 없음 — 기여도 비례가 자연스러운 상한
- 위협(threat) 시스템 자체가 없으므로 "어그로 끌기" 트롤도 불가능

### 10.4 봇 재시작
- 진행 중 레이드: 시작 시 active 행을 로드, 임베드 message_id 로 메시지 재첨부
- 메시지가 삭제됐다면 새 메시지 생성하고 raids.message_id 갱신
- 만료 시간이 지났다면 그 즉시 종료 처리

### 10.5 페어플레이
- 오너의 `/레이드취소` 만 강제 종료 가능
- 오너 무시 모드 유저는 참가 불가 (정책 일관성)

## 11. 확장 아이디어

이번 v1 에서는 제외하되, 향후 단계에서 검토:

### 11.1 도파민 강화
- **연속 처치 보너스**: 한 주 내 N마리 연속 잡으면 길드 전체 XP 부스트 24시간
- **보스 도감**: 처치한 보스 컬렉션 임베드 (`/도감`)
- **칭호 시스템**: 첫 처치자 / 결정타 누적 / TOP1 횟수에 따라 칭호 부여, 닉네임 옆 이모지로 표시
- **시즌**: 분기마다 보스 라인업 변경 + 시즌 한정 칭호

### 11.2 협동 강화
- **파티 매칭**: 작은 파티(3~5명) 미니 보스
- **공조 스킬**: 두 명이 같은 턴에 사용 시 추가 데미지
- **부활/지원 스킬**: 다른 참가자의 쿨다운/마나 회복

### 11.3 경쟁 강화
- **레이드 랭킹**: 보스별 최단 처치 / 최대 단일 데미지 / 결정타 횟수
- **명예의 전당** 임베드 (`/명예의전당`)

### 11.4 경제 도입
- **골드** 재화 추가 → 상점 → 비전투 부스트/장비
- 골드 도입은 큰 변경 — 별도 기획서 필요

### 11.5 등장 트리거 다양화
- 정기 스케줄(`raid_schedule`)
- 길드 누적 XP 임계점 달성 시 자동 등장 ("길드의 활기가 보스를 깨웠다")
- 특정 이모지 반응 누적 시 ("도전장이 100개 쌓이면 등장")
- 이벤트 보스: 할로윈/크리스마스/봇 생일

### 11.6 빌드 시너지
- 스탯 빌드별 시너지 효과:
  - **풀 STR**: 평타 폭딜형
  - **풀 AGI**: 빠른 평타 + 치명 누적
  - **풀 INT**: 스킬 연발형
  - **풀 LUK**: 도박형 (대미지 분산 크지만 잭팟 가능)
- 스탯 빌드 추천 임베드 `/빌드추천` (지능 위주, 행운 위주 등 템플릿)
