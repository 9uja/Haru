# HaruBot

단일 디스코드 서버(길드) 전용 봇. Python + [discord.py](https://github.com/Rapptz/discord.py) 기반이며,
슬래시 커맨드로 동작합니다.

## 기능

| 명령어 | 설명 | 권한 |
| --- | --- | --- |

| `/전체확인` | 전체 멤버 음성 활동을 임베드로 조회 (◀▶ 페이지·🔃 정렬) | 서버 관리 |

| `/청소 <개수>` | 이 채널의 최근 메시지 N개(1~100) 일괄 삭제 — 고정 메시지는 보존 | 메시지 관리 |
| `/레벨 [멤버]` | 본인/지정 멤버의 레벨·XP·진행률·서버 순위·활성 부스트 | 누구나 |
| `/랭킹` | 서버 XP 상위 10명 임베드 | 누구나 |

| `/레벨설정 [메시지xp] [음성xp] [알림채널] [알림채널해제]` | 레벨링 옵션 변경 | 서버 관리 |
| `/능력치 [멤버]` | RPG 스탯 시트 (힘/민첩/지능/행운 + 효과 요약) | 누구나 |
| `/능력치분배 <스탯> <포인트>` | 미분배 포인트를 스탯에 할당 | 누구나 |

| `/레이드참가` | 진행 중인 레이드 참가 + 본인 ephemeral 패널(상태창 + 평타 + 4스킬 버튼 + 마나바) | 누구나 |
| `/레이드소환 <보스>` | 보스 즉시 소환 (🔥 일반·❄️ 엘리트·🌑 레이드 3종) | 서버 관리 |
| `/레이드채널설정 [채널]` | 전용 채널 지정 — 생략 시 `보스레이드` 자동 확보/생성 | 서버 관리 |
| `/레이드취소` | 진행 중인 레이드 강제 종료 | **오너 전용** |
| `/채널안내` | 공개 채널과 설명(토픽)을 안내 | 누구나 |
| `/환영채널설정 [채널]` | 신규 입장 환영 메시지를 보낼 채널 지정 | 서버 관리 |
| `/범프채널설정 [채널]` | DISBOARD 범프 리마인더를 보낼 채널 지정 | 서버 관리 |
| `/범프알림` | `범프알림` 역할을 받거나 해제(본인) — 알림 시 역할 멘션 | 누구나 |
| `/기억추가 <내용>` | 서버 전체가 참고할 지식 저장(관리자) | 서버 관리 |
| `/기억목록` · `/기억삭제 <번호>` | 서버 지식 보기 / 삭제 | 서버 관리 |
| `/내기억추가 <내용>` · `<봇이름>야 … 기억해` | 내 개인 기억 저장(내 대화에서만 참고) | 누구나 |
| `/내기억목록` · `/내기억삭제 <번호>` | 내 개인 기억 보기 / 삭제 | 누구나 |
| `<봇이름>야 <메시지>` | 봇 닉네임 + 호격으로 시작하면 AI가 대화로 응답 (Gemini) | 누구나 |
| `<봇이름>야 번역 <문장>` | 한국어↔영어 자동 번역 (`… 번역 일본어 …` 로 지정 가능) | 누구나 |
| `/관리 db초기화 <범위>` | 영역별 DB 초기화 (chat·memory·voice·warnings·all) — 버튼 확정 | **오너 전용** |
| `/관리 무시 <유저>` · `/관리 무시해제 <유저>` | AI가 해당 유저 메시지를 완전히 무시 | **오너 전용** |
| `/관리 지시 <유저> <지시>` · `/관리 지시해제 <유저>` | 해당 유저 대화에 추가 system 지시 주입 | **오너 전용** |
| `/관리 목록` | 적용된 무시/지시 오버라이드 보기 | **오너 전용** |

> 모든 슬래시 명령은 **한국어 이름**만 제공됩니다(영어 이름 제거됨).
> AI 대화/번역은 슬래시가 아니라 채팅에 `<봇이름><야|아> …` 로 말하면 됩니다.
> **트리거는 봇의 길드 닉네임에 따라 자동 결정**됩니다 — 봇 이름이 `하루` 면 `하루야`, `준` 이면 `준아`(받침 자동 판별).
> 서버에서 봇을 다른 이름으로 바꾸기만 해도 즉시 새 트리거가 적용됩니다(코드 수정/재배포 불필요).
> 그 외에 봇이 **낮은 확률(기본 2%)로 일반 채팅에 가볍게 끼어들어 답장**합니다(`REPLY_CHANCE` 로 조절·`0`이면 끔, 쿼터 보호용 전역 쿨다운 있음).
> AI 기억은 **서버 지식(관리자 `/기억추가`, 전체 공유)** 과 **개인 기억(누구나 `<봇이름>야 … 기억해`/`/내기억추가`, 본인 대화에만)** 으로 분리됩니다. 모델 학습이 아니라 기억을 프롬프트에 주입하는 방식입니다.
> 대화 맥락은 **채널별 + 유저별**로 함께 기억(DB 저장, 재시작 유지)하며 턴 수는 `CHAT_HISTORY_TURNS`(기본 8)로 조절. 기록이 `CHAT_HISTORY_MAX_ROWS` 를 넘으면 **오래된 순으로만 자동 정리**(평소엔 최대 보존).
> '휴면' 역할은 해당 멤버가 음성에 다시 입장하면 **자동 해제**됩니다.

### 음성 활동 추적 & 비활성 안내
- 음성 채널 입장/퇴장을 로그 채널에 기록하고, 멤버별 마지막 활동 시각·누적 체류시간을 **PostgreSQL** 에 영속화.
- 서버(길드) 입·퇴장(나갔다 재입장)을 감지해 멤버별 입·퇴장 횟수를 누적 기록(`/스탯` 으로 조회).
- 한 달(기본 30일) 이상 음성 활동이 없는 멤버를 `/활동확인` 또는 **자동 정기 보고**(기본 매주)로 안내.
- 안내 목록은 `@username`(클릭 가능·무음 멘션) 형식이라 관리자가 바로 상호작용 가능.

### 온보딩
- 신규 멤버 입장 시 환영 채널(미설정 시 서버 시스템 채널)에 **랜덤 인사** + 채널 안내 힌트.
- `/채널안내` 로 공개 채널과 각 채널 설명(토픽)을 한눈에 안내.

### 레벨링 · 아이템 · 부스트
- 메시지 XP: **1 / 메시지** (고정), 유저당 60초 쿨다운, 봇/명령(`!?/.` 시작) 제외.
- 음성 XP: **5분마다 60** (=시간당 720), 같은 채널 **비봇 2명 이상** + **AFK/청각차단 아님** 조건.
- 메시지/음성 XP 각각 `/레벨설정 메시지xp:False` / `/레벨설정 음성xp:False` 로 끌 수 있음.
- 레벨업 안내 채널 지정: `/레벨설정 알림채널:#level-up` (미지정 시 발생 채널).
- 레벨 곡선: **메이플스토리 영감** `xp_to_next(L) = 0.5L²+5L+10` — L1=15, 최대 **Lv 999**.
  누적 XP 에서 매번 산정(저장 X).
- **아이템 드롭**: 메시지 XP 적립 시 **2% 확률**로 4종 중 하나 가중치 추첨.
  - 🍙 작은 비상식량(+50% / 30분), 🍱 큰 비상식량(2배 / 20분),
  - 🧪 엘릭서(3배 / 10분), 🍀 행운의 부적(5배 / 5분, 희귀).
- 사용 시 **활성 부스트**로 등록 — 유저당 1개, 새 사용은 기존을 덮어쓴다.
  부스트 적용 중엔 메시지·음성 XP 모두 곱연산.
- Cloudflare 1015 가드 활성 시 송신만 스킵, DB 누적은 계속.

### RPG 스탯 (보스 레이드 대비)
- 레벨업 시 **4 포인트** 자동 획득, 자유 분배.
- 스탯 4종:
  - 💪 **힘** — 데미지 _(보스 레이드용)_
  - 💨 **민첩** — 쿨타임 감소, 치명타 데미지 _(보스 레이드용)_
  - 🧠 **지능** — 마나 재생, 약점 공격 확률 _(레이드용)_ · **경험치 획득 +0.5%/pt**
  - 🍀 **행운** — 치명타 확률 _(레이드용)_ · **아이템 드롭 +1%/pt**
- 분배 이력(`stat_allocations`) 을 이벤트 단위로 LIFO 기록 → **레벨 손실 시 가장 최근 분배부터 정확히 환불**.
  1레벨이든 100레벨 손실이든 동일 로직 (잃은 레벨 × 4 포인트만큼 환수).
- 도박/감소 등 향후 기능은 `Leveling.lose_xp(guild_id, user_id, amount)` 한 줄만 호출하면 됨 — XP 차감·레벨 비교·환불까지 자동.

### 봇 오너(최고 등급) 전용 기능
- **고정 디스코드 ID 한 명**(`owner.py` 의 `OWNER_ID`, 기본 `379934490866352130` / 닉네임 `구자`) 만이 사용 가능.
  서버의 관리자 역할 권한과 **별개**로, 봇 정책을 바꾸는 명령은 이 한 명만 받는다.
- `/관리` 슬래시 그룹: `db초기화`(버튼 확정), `무시`/`무시해제`, `지시`/`지시해제`, `목록`.
- AI 자연어 명령도 동일 권한: `<봇이름><야|아> …` 로 입력하면 됨(예시 아래에선 `하루야` 기준).
  - `하루야 DB 초기화 chat` 같은 형태는 안전을 위해 `/관리 db초기화` 슬래시 확정 흐름으로 안내합니다.
  - `하루야 <@유저> 무시` / `하루야 구자 무시 해제` — 멘션·디스코드 ID·닉네임 모두 인식.
  - `하루야 <@유저> 에게는 존댓말로만 짧게 답해` — 그 유저와의 대화에만 적용되는 추가 지시.
- 무시(`ignore`) 모드 유저는 AI가 트리거/임의 답장/기억 모두 응답하지 않음. 오너 본인은 절대 무시 대상이 될 수 없도록 잠금.

### 범프 리마인더
- `/범프채널설정` 으로 알림 채널을 지정해 두면, DISBOARD `/bump` **성공을 감지**해
  **2시간 뒤 그 채널에 알림**. 봇 재시작에도 예약 유지.
- `/범프알림` 을 누르면 `범프알림` **전용 역할**이 부여/해제되며, 리마인더는
  그 역할 하나만 멘션해 보냅니다(개별 유저 멘션 배치가 필요 없어 깔끔).
- 봇에 **역할 관리** 권한이 필요하며, 봇 역할이 `범프알림` 역할보다 위에 있어야 합니다.
- ※ 봇이 `/bump` 를 대신 실행하는 건 Discord 정책상 불가 → 사람이 누르도록 알림만 보냄.

## 빠른 시작

```powershell
# 1. 의존성 설치 (가상환경 권장)
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt

# 2. 환경 변수 설정 (토큰, 길드 ID, DATABASE_URL 입력)
Copy-Item .env.example .env

# 3. 실행
.\.venv\Scripts\python.exe bot.py
```

> 음성 활동 저장에 PostgreSQL이 필요합니다. 무료 [Neon](https://neon.tech) DB 한 개면 충분합니다.
> 토큰 발급, DB 준비, 인텐트/권한, 봇 초대 방법은 [docs/SETUP.md](docs/SETUP.md)를 참고하세요.
